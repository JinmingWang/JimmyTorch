# JimmyTorch

Personal PyTorch framework for deep learning research.


## Project Structure

A JimmyTorch project *must* follow this structure:

```
ProjDir/
├── Datasets/               # Dataset implementations
├── JimmyTorch/*            # JimmyTorch library code, read-only
├── Models/                 # Model implementations
├── Reports/*               # Generated reports, logs, figures, related code
├── Runs/                   # Auto-generated: logs, checkpoints, results
├── TrainEvalTest/          # Training orchestration code
└── main_*.py               # Entry scripts (train/test)
```

In the following we explain each component.

The structure of `Datasets/` *must* be:

```
Datasets/
├── __init__.py         # Must be a package
├── MyDataset_1.py      # Example dataset, rename for your project
├── MyDataset_2.py
└── ...
```

The `JimmyTorch/` folder contains the library code, read-only.

The structure of `Models/` *must* be:

```
Models/
├── __init__.py         # Must be a package
├── Method_1/           # Example method, rename for your project
│   ├── __init__.py     # Must be a package
│   ├── M_1_v_1.py      # Example model variant, rename for your project
│   ├── M_1_v_2.py
│   ├── ...             # Additional variants
│   └── components.py   # Put all submodules here, model variants only implement the main class
├── Method_2/
└── ...
```

The `Reports/` folder is for any result generation code, figures, tables, etc.

The `Runs/` folder is auto-generated by JimmyTorch during training, it has the following structure:

```
Runs/
├── MyDataset_1/
│   ├── Method_1/
│   │   ├── train_1/
│   │   ├── train_2/...
│   │   └── ...
│   ├── Method_2/...
│   └── ...
├── MyDataset_2/...
└── ...
```

Where each train folder contains 1) `best.pth` - best checkpoint, 2) `last.pth` - latest checkpoint, 3) `comments.txt` - training comments and hyper-params, 4) `events.out...` - Tensorboard logs, 5) `model_arch.txt` - model architecture printout, 6) `runtime_param_buffer.yaml` - for hot-reloading params during training, 7) `log.txt` - LLM readable training log, and 8) any generated test results (e.g. `results.csv`).

The `TrainEvalTest/` folder contains the training orchestration code, including the `Experiment` class which manages the entire lifecycle of an experiment, and the `Trainer` class which implements the training loop. The `main_*.py` files are the entry points for training and testing, where you configure your experiment and start the process.
